MCP Hangar Operator has been installed.

{{- if gt (int .Values.replicaCount) 1 }}
NOTE: replicaCount is set to {{ .Values.replicaCount }} (HA mode). The operator
gates /readyz on leader election, so only the elected leader Pod reports
Ready; the other {{ sub (int .Values.replicaCount) 1 }} replica(s) will show
NotReady in `kubectl get pods` until they win leadership on a failover. This
is expected. Avoid `helm upgrade --wait` with replicaCount >= 2 -- Helm waits
for every Pod in the Deployment to become Ready, which standby replicas never
do on their own.
{{- end }}

Check CRD status:

  kubectl get mcpservers
  kubectl get mcpservergroups
  kubectl get mcpdiscoverysources

View operator logs:

  kubectl logs --namespace {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }} -f

NOTE: CRDs are not removed when you run `helm uninstall`.
To remove CRDs manually:

  kubectl delete crd mcpservers.mcp-hangar.io
  kubectl delete crd mcpservergroups.mcp-hangar.io
  kubectl delete crd mcpdiscoverysources.mcp-hangar.io
